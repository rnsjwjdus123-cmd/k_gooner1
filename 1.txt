# 🛠️ Git 생태계 한눈에 비교하기

| 구분 | 도구/서비스 | 주요 역할 | 한 줄 요약 |
| :--- | :--- | :--- | :--- |
| **핵심 기술** | **Git** | 내 컴퓨터의 코드 변경 이력 기록 (로컬) | "코드 타임머신" |
| **저장 공간** | **GitHub** | 온라인에 코드를 올리고 공유 (원격) | "개발자들의 성지/주차장" |
| **GUI 도구** | **Git Desktop** | 버튼 클릭으로 Git을 쉽게 조작 | "명령어 몰라도 OK" |
| **편집기 내장** | **VS Code** | 코드 짜면서 바로 버전 관리 가능 | "가장 편리한 작업대" |

---

## 💡 주요 명령어 및 기능 비교

### 1. Git (터미널 명령어)
* `git add`: 저장할 파일을 고르기 (장바구니 담기)
* `git commit`: 현재 상태 저장하기 (사진 찍기)
* `git push`: 온라인(GitHub)에 올리기

### 2. GitHub (웹사이트 기능)
* `Repository`: 프로젝트 저장소 (폴더)
* `Pull Request (PR)`: 내가 수정한 코드를 합쳐달라고 요청하기
* `Issues`: 버그나 할 일 기록하기

### 3. VS Code 소스 제어 (왼쪽 3번째 아이콘)
* 파란색 숫자가 뜨면 수정된 파일이 있다는 뜻입니다.
* `+` 버튼을 누르면 `add`, 메시지 적고 `V` 버튼 누르면 `commit`입니다.


수정하는 중입니다.


# 🚀 나의 첫 Git & GitHub 연동 정리 (2026.01.15)

현재 내 화면은 **로컬(내 컴퓨터)**에서 만든 코드를 **원격(GitHub 온라인)**으로 보내기 위한 설정 단계입니다.

## 1. 현재 화면의 주요 구성 요소 설명
* **VS Code (왼쪽 배경)**: 실제 코드를 수정하고 저장하는 장소입니다. `git.test.txt` 파일을 수정 중인 것이 보입니다.
* **GitHub Desktop (가운데 팝업)**: 어려운 명령어 대신 클릭으로 GitHub에 코드를 올리게 도와주는 도구입니다.
* **Publish Repository (중앙 버튼)**: 내 컴퓨터에만 있는 이 폴더를 GitHub 사이트에 새로운 저장소로 만들겠다는 뜻입니다.

## 2. 작업 흐름 (Workflow) 이해하기
내 코드가 온라인에 올라가기까지의 3단계입니다.

1. **수정 (Modify)**: VS Code에서 파일을 고치고 저장합니다.
2. **커밋 (Commit)**: `Summary` 칸에 "git설명"이라고 적으신 것처럼, 변경 사항에 이름을 붙여 내 컴퓨터에 저장합니다.
3. **푸시 (Push/Publish)**: `Publish repository` 버튼을 눌러 GitHub 온라인 저장소로 최종 전송합니다.

## 3. 화면 속 설정값의 의미
* **Name**: `k_gooner` (GitHub에 생길 저장소의 이름입니다.)
* **Description**: `dkanrjsk` (이 프로젝트가 무엇인지 설명하는 칸입니다.)
* **Keep this code private**: 체크되어 있으면 나만 볼 수 있는 비공개 저장소가 됩니다.

## 4. 꼭 기억할 Git 핵심 단어
| 용어 | 의미 | 비유 |
| :--- | :--- | :--- |
| **Repository** | 프로젝트 저장소 | 온라인 주차장 |
| **Commit** | 현재 코드 상태 저장 | 타임머신 기록 포인트 |
| **Push** | 온라인으로 코드 보내기 | 서버에 업로드 |
| **Pull** | 온라인에서 코드 가져오기 | 최신판 다운로드 |

---
> **Tip**: 내일 시험에서 깃 명령어가 나온다면 `git push`가 바로 이 화면의 `Publish` 버튼과 같은 역할을 한다는 점을 기억하세요!



# 🌐 GitHub 업로드 완료 및 온라인 확인 정리 (2026.01.15)

이 문서는 내 로컬 프로젝트를 **GitHub 원격 저장소**에 성공적으로 올린 과정을 기록합니다.

## 1. GitHub Desktop을 통한 배포 (Publish)
* **화면 상황**: `Publish repository` 팝업에서 `k_gooner`라는 이름으로 저장소를 생성했습니다.
* **동작**: 파란색 버튼을 누르는 순간, 내 컴퓨터의 `git.test.txt`를 포함한 모든 파일이 GitHub 서버로 전송됩니다.

## 2. GitHub 웹사이트에서 확인하기
보내주신 브라우저 화면(`image_4ebc8a.png`)은 내 온라인 계정에 저장소가 잘 만들어졌는지 확인하는 화면입니다.

* **저장소 목록 (Repositories)**:
    1. **k_gooner (사설)**: 방금 올린 프로젝트입니다. '사설(Private)'로 설정되어 나만 볼 수 있습니다.
    2. **git_tast (공공)**: 이전에 만든 테스트용 저장소입니다. '공공(Public)'으로 되어 있어 누구나 볼 수 있습니다.
* **상태**: `2 minutes ago 업데이트`라는 표시를 통해 방금 막 업로드가 성공했음을 알 수 있습니다.

## 3. 파일 누락 해결 팁 (이미지 에러 관련)
현재 화면 중 `denji.jpg` 로딩 에러나 웹페이지의 엑박(이미지 깨짐) 현상은 다음 이유 때문입니다.

* **현상**: 터미널에 `GET /static/tanjiro.jpg 404` 에러 발생.
* **원인**: `app.py` 코드에는 `tanjiro.jpg`라고 썼지만, 실제 폴더에는 `deng.jpg`나 `loopy.jpg`처럼 이름이 다르게 저장되어 있습니다.
* **해결**: 반드시 **코드상의 이름**과 **파일의 실제 이름**을 100% 일치시켜야 GitHub에 올린 후에도 이미지가 잘 보입니다.

## 🚀 앞으로의 작업 흐름
파일을 수정했다면 아래 3단계를 반복하면 됩니다.

1. **Modify**: VS Code에서 코드 수정 후 저장 (Ctrl + S)
2. **Commit**: GitHub Desktop 왼쪽 하단에 수정 내용을 적고 `Commit to main` 클릭
3. **Push**: 상단의 `Push origin` 버튼을 눌러 온라인에 반영
